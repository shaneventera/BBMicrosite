Design Challenge API

Ventera has created this site to help guide you in using the API for the design challenge.

Description

The BB 2.0 design challenge API supports querying Medicare Part B “carrier” claims data. The API will return Explanation of Benefits records for beneficiaries grouped by provider (NPI ID).

Note that for the design challenge the response has been limited to 20 fields which were prioritized based on requirements specified in the design challenge readme as well as on what a submitting provider would need to link returned data to submissions and for verification.

Authentication

Using the App

Step 1

Once the application is started, we will need to ensure that we are authenticated with the server before we can complete anything. First we will run a test to see if the app is up and we get an unauthorized error. Run:

curl http://localhost:8080/resources/trusted_client

You should expect some response with a JSON object with an error: "Unauthorized" field.

Step 2

We need to get authorization before proceeding.

To authenticate against the server run the curl command:

curl -v http://localhost:8080/oauth/token \
   -H "Accept: application/json" \
   -H "Accept-Language: en_US" \
   -u "trusted-app:secret" \
   -d "grant_type=client_credentials"

You should expect a large payload with a field in the JSON response titled: access_token:

Step 3

Copy the complete value of the access_token and paste it into this command: Make sure to replace {AUTHORIZATION_TOKEN} with the copied value.

curl -H "Authorization: Bearer {AUTHORIZATION_TOKEN}" http://localhost:8080/resources/trusted_client

If it was successful you should see the response: hello user authenticated by trusted client

Running Automated Tests

Automated Tests can be run from this directory using:

./test.sh

Running the Query Script

The Query script can be run using the following format:

./query.sh NPI_VALUE

Where the NPI_VALUE is the npi you are searching for
